# R8 v2 設計ドキュメント
作成：2026-04-20
ステータス：設計段階（実装はv1論文完成・coconala出品後）

---

## 概要

v1（r8.py v16）は語彙密度スクリーニング層として機能する第一段階。
v2は「第二検出層」として構造パターン検出を実装し、v1のFalseNegativeを可視化する。

**設計原則：**
- v1の辞書を拡張するのではなく、v2は独立した第二層として設計する
- 語彙密度で検出できないものを構造・シーケンス・共起で検出する
- 実装前に設計ドキュメントを確定する（このファイルがその役割）

---

## アーキテクチャ全体像（多段階）

```
Stage 1（v1）：語彙密度スクリーニング
　LOW確定 → 除外
　MEDIUM/HIGH → Stage 2へ
　LOW + 特定共起パターン → Stage 2へ（例外ルート）

Stage 2（v2）：構造パターン検出
　共起パターン重みづけ
　段落シーケンス分析
　ScarcityRisk複合検出
　GCI（目的収束率）スコアリング

Stage 3（v3以降）：非線形モデル・CVI統合
　シグモイド変換
　CMI×CVI交互作用モデル
　集合的拡散リスク予測（Granovetter接続）
```

## v2 第二層：設計標的

### 1. 段落シーケンス検出（最優先）

操作的テキストの核心は語彙の密度ではなく段落の順序にある。

**基本シーケンス（Cialdini 1984 Ch.3 接続）：**
```
否定・問題提起（あなたも悩んでいますよね）
　↓
共感・類似性確立（私も同じだった）
　↓
小さなコミットメント要求（まず読んでみてください）
　↓
価値観の再定義（本当の自分とは）
　↓
行動誘導（今すぐ）
```

**与沢翼「秒速で一億円稼ぐ条件」で確認された典型パターン（Gemini分析）：**
```
否定（世間の常識を否定）
→ 再定義（著者理論の提示）
→ 具体例（自身の経験）
→ 普遍化（全ビジネスに応用可能な法則に抽象化）
→ 危機感の醸成（現状維持のリスク）
→ 解決策の提示（特定の行動を推奨）
```

**理論的根拠：**
- Cialdini（1984）第3章：コミットメントと一貫性（p119・p123・p136・p161）
- Cialdini（1984）第4章：社会的証明（p189・p228・p263）
- Baumeister & Leary（1995）：帰属欲求の段階的活用

**実装方針：**
- 段落単位での役割分類（問題提起／共感／要求／誘導）
- シーケンスの出現順・密度・間隔をスコア化
- 単独段落では検出しない（FPリスク回避）

---

### 2. コミットメント取得シグナル検出

**理論的根拠（Cialdini p181）：**
最初のコミットメントを取った後、語彙密度を下げて「自然な流れ」として誘導する。
v1はその「自然な流れ」部分を低リスクと判定する。

**検出標的（5類型）：**
1. 同意誘導型文末：「〜ですよね」「〜と思いませんか」「〜に気づいていますか」
2. 選択肢の偽装：「AかBか、あなたが決めてください」
3. 小さな行動の要求：「まず無料で試してみてください」「LINEを追加するだけ」
4. 読者への問いかけ連続（段落内疑問形密度）
5. 自己開示の促進：「あなたの悩みを教えてください」

**注意：** 単独では全てFPリスク高。構造パターンとして設計（複数共起＋後続誘導語彙との連鎖）。

---

### 3. ScarcityRisk 独立カテゴリ化

現行v1ではFearRisk・AbsolutistWords・ClickbaitRiskに分散。v2で統合・独立化。

**5類型（Brock 1968・Brehm 1966 理論的根拠）：**
1. 時間的希少性：「今日限り」「あと〇時間」「締め切り」
2. 数量的希少性：「残り3席」「限定50名」「在庫わずか」
3. 情報的希少性：「一部の人しか知らない」「非公開情報」
4. 関係的希少性：「あなただけに」「特別に選ばれた方へ」
5. 機会的希少性：「この機会は二度と来ない」「人生最後のチャンス」

**重要：** 5類型の共起が増幅効果を生む（Fromkin & Brock 1971の×6実験）。
線形加算ではなく共起重みづけスコアリングが必要。

---

### 4. 帰属の非対称性検出（Cialdini p316・Weiner 1986）

```
集合的代名詞 + 成功語彙の共起：「私たちは〜を達成した」
個人代名詞 + 失敗・責任語彙の共起：「あなたが〜できなかった理由」
```

**理論的根拠：**
- Cialdini（1984）p316：勝利時は「俺たち（We）」・敗北時は「彼ら（They）」
- Weiner（1986）帰属理論：成功の内的帰属＋失敗の外的帰属
- ブラック企業構造・MLM構造の典型パターン

---

### 5. 苦痛再定義語彙（Cialdini p144・p147）

渡邉・Makoなり・与沢コーパスのLogicalRisk 1.0問題の直接的原因。

**検出標的：**
- 「つらいのは成長の証」
- 「逃げる人間は最初から要らない」
- 「感謝できない人は伸びない」
- 「素直になれ」「抵抗感こそが成長の証」
- 「我々だけが知っている」（Janis p23：反意見の封鎖と接続）

**理論的根拠：**
- Cialdini（1984）第3章p144・p147：苦痛体験を自己成長として再定義
- Nakane（1967）縦社会：苦痛→忠誠のサイクルを文化的に強化
- AuthorityRisk/EnemyFrame境界に位置するカテゴリ

## GCI（Goal Convergence Index・目的収束率）

**概念：**
操作者の目的（入信・購買・会員登録等）を文脈から仮想的に推定し、
そのテキストが目的に向かって収束している度合いをスコア化する。

**設計方針：**
1. 目的カテゴリを事前定義：
   - 金銭的：投資・購入・課金
   - 社会的：入会・入信・コミュニティ参加
   - 関係的：個人情報提供・個別相談・LINEへの誘導
   - 認知的：信念変容・世界観の再定義
2. 各目的カテゴリに対応する語彙クラスターを設計
3. 段落末尾からの逆算シーケンスで収束率を測定
4. CMI×CVI交互作用モデルの第三次元として統合

**理論的根拠：**
- Kahneman（2011）p204-205：frame manipulation = 意思決定の結果を変える語彙的手法
- Eccles & Wigfield（2002）EVT：目的関数の操作が読者の動機を変える
- Winnicott「問題行動はあきらめの反対」：潜在的期待値の再活性化

**実装優先度：** Stage 2後半。段落シーケンス検出が安定してから。

## CMI×CVIモデル（Stage 3設計）

```
操作リスク = CMI + CVI + β(CMI × CVI)
β > 0 ：相乗効果（巧みな操作者はβを最大化する）
β < 0 ：相殺効果
```

**CVI構成要素候補：**
- 一貫性傾向（年齢との相関：Cialdini p177）
- 権威服従傾向（Altemeyer 1981 RWA尺度）
- 不確実性耐性
- 認知的流暢性感受性（Zajonc 1968 mere exposure effect）
- 類似性感受性

**非線形モデル候補定式化：**
```
CMI_nonlinear = 1 / (1 + e^(−k(CMI_linear − θ)))
θ：個人応答閾値（CVIが規定）
k：遷移勾配パラメータ
```

**制約：** 現行224件・単一評価者では θ・k の推定不可能。
Stage 3は行動アウトカムデータ（被害額・入会率等）が利用可能になってから。

**Granovetter接続（1978）：**
- 個人内閾値：CVIが閾値を規定→CMIが超えた瞬間System 1が支配的に
- 集団内閾値分布：操作的テキストが閾値分布の最低層を動かす→連鎖拡散
- 論文Future Work 7.4に記述済み

## 内向きベクトル・ベクトル方向性分析（Stage 2-3）

**概念：**
```
外向きベクトル：市民・消費者・有権者への印象操作
内向きベクトル：組織内部・自己への認知操作（「自分たちは間違っていない」）
自己向きベクトル：発信者自身への認知的不協和解消
```

**理論的根拠：**
- Festinger（1957）認知的不協和：内向き自己操作は外向き説明と矛盾するほど強化される
- 官僚・企業の危機対応テキストは3方向に同時に機能する

**コーパス設計への示唆（Phase 2）：**
- 財務省統計書き換え関連文書・旧ビッグモーター会見等の追加
- human_labelに「ベクトル方向」列の追加検討
- CMIが低く出るが human_label HIGH → FalseNegativeの典型事例として記録

**語彙候補（FPリスク高・v2文脈検出で設計）：**
- 「適切に対応した」「法令に従い」「遺憾である」「真摯に受け止める」
- これらは外部説明ではなく内部規範再確認の機能を持つ可能性あり
- 単独辞書追加は禁止（FPリスク）・共起パターンで条件付き

## 論文に入れない・設計ドキュメント記録のみの概念

**Dutton & Painter（1981）トラウマ・ボンディング：**
- 間欠強化の構造的類比：報酬と罰の非規則的交互提示が依存を強化
- テキスト的実装：承認語彙と批判語彙の交互配置
- 論文には不記載（臨床文脈が強すぎる）・v2設計の参照概念として保持

**情報カスケード（Bikhchandani et al. 1992）：**
- Granovetter接続の補足理論
- 個人が自分の信念より他者の行動を優先する条件
- Stage 3モデルの理論的背景として保持

**Winnicott「問題行動はあきらめの反対」：**
- GCIの潜在的期待値再活性化メカニズムの説明概念
- 論文では概念言及のみ・直接引用は避ける
