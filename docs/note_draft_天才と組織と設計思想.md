【思考素材：天才・組織・設計思想】

設計者と実装者は分離できる。トーバルズは両方やったが、ゲイツはカトラーを、ジョブズはウォズニアックを必要とした。一人で全部やる必要はない。
官僚化した組織は「どちらが正しいか」ではなく「どちらがリスクが少ないか」で判断する。ソニーがAppleを門前払いしたのは合理的判断だったが、その合理性が致命傷になった。
扱いにくい天才を左遷するか窓際にするかしかできない組織は、その人材のアップサイドをゼロにする。外に出してリスクゼロでアップサイドだけ取る発想が日本企業にはなかった。
推論ベースの思考は間違いを含むが、推論しない人間は間違いも起こさない。何も生まないからだ。
同じ源流（ゼロックスのGUI）から出発してジョブズは美学の追求へ、ゲイツは普及の追求へ向かった。起点が同じでも思想が結果を決める。

「AIを使えなければ終わり」という焦りは本物か。しかしその「使える」とは何を指しているのか。
2. 本当に必要な能力とは何か
コードでも数学でもない。問題を構造化して言語化する能力。グライダーとエンジンの話。
3. 私の実験
非エンジニアがAIと対話しながら認知リスク分析ツールを作った記録。何ができて何ができなかったか。
4. 再現可能性について
これは特別な話ではない。条件と能力の話だ。
